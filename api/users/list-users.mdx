---
title: 'List Users'
api: 'GET /api/management/users'
description: 'Retrieve a paginated list of all users in the system'
---

## Endpoint

```text
GET /api/management/users
```

## Authentication

This endpoint requires authentication with a valid JWT token and the **ADMIN** role.

```text
Authorization: Bearer <your_jwt_token>
```

## Query Parameters

<ParamField query="page" type="integer" default="0">
  The page number to retrieve (zero-indexed)
</ParamField>

<ParamField query="size" type="integer" default="20">
  The number of users per page (maximum: 100)
</ParamField>

<ParamField query="sort" type="string" default="id">
  The field to sort by. Can include direction with comma separator (e.g., `id,desc`)
  
  Available sort fields:
  - `id`
  - `email`
  - `firstName`
  - `lastName`
</ParamField>

## Response

Returns a paginated list of users wrapped in the standard `ApiResponse` format.

<ResponseField name="success" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="timestamp" type="string" required>
  ISO 8601 timestamp of the response
</ResponseField>

<ResponseField name="data" type="object" required>
  Paginated user data
  
  <ResponseField name="data.content" type="array" required>
    Array of user objects
    
    <ResponseField name="content[].id" type="integer" required>
      Unique identifier for the user
    </ResponseField>
    
    <ResponseField name="content[].email" type="string" required>
      User's email address (unique)
    </ResponseField>
    
    <ResponseField name="content[].firstName" type="string" required>
      User's first name
    </ResponseField>
    
    <ResponseField name="content[].lastName" type="string" required>
      User's last name
    </ResponseField>
    
    <ResponseField name="content[].roles" type="array" required>
      Array of role strings assigned to the user. Available roles: `MEMBER`, `ADMIN`
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="data.pageable" type="object" required>
    Pagination metadata
    
    <ResponseField name="pageable.pageNumber" type="integer">
      Current page number (zero-indexed)
    </ResponseField>
    
    <ResponseField name="pageable.pageSize" type="integer">
      Number of items per page
    </ResponseField>
    
    <ResponseField name="pageable.offset" type="integer">
      Offset of the first item in the current page
    </ResponseField>
  </ResponseField>
  
  <ResponseField name="data.totalPages" type="integer" required>
    Total number of pages
  </ResponseField>
  
  <ResponseField name="data.totalElements" type="integer" required>
    Total number of users across all pages
  </ResponseField>
  
  <ResponseField name="data.size" type="integer" required>
    Number of items in the current page
  </ResponseField>
  
  <ResponseField name="data.number" type="integer" required>
    Current page number (zero-indexed)
  </ResponseField>
  
  <ResponseField name="data.first" type="boolean" required>
    True if this is the first page
  </ResponseField>
  
  <ResponseField name="data.last" type="boolean" required>
    True if this is the last page
  </ResponseField>
  
  <ResponseField name="data.empty" type="boolean" required>
    True if the page contains no items
  </ResponseField>
</ResponseField>

## Error Responses

### 401 Unauthorized

Returned when authentication credentials are missing or invalid.

```json
{
  "success": false,
  "timestamp": "2026-03-03T10:30:00Z",
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Authentication required"
  }
}
```

### 403 Forbidden

Returned when the authenticated user does not have the ADMIN role.

```json
{
  "success": false,
  "timestamp": "2026-03-03T10:30:00Z",
  "error": {
    "code": "FORBIDDEN",
    "message": "Access denied. ADMIN role required."
  }
}
```
